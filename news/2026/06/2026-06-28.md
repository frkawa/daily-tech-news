### ★★★☆☆ [hoge+fuga@example.com は「エイリアス」ではない](https://zenn.dev/mai_llj/articles/3eefb32e632563) `メール`

- Gmail等で使用される「hoge+fuga@example.com」という表記は、エイリアスではなく「アドレス拡張（address extension）」という機能であり、これを誤って理解している人が多い
- メールアドレスの@左側（local-part）はサーバ側が自由に解釈できる部分で、+記号の扱いもRFC仕様では決まっておらず、受信サーバの設定に依存する
- 本来のエイリアスはSendmailの/etc/aliasesを指し、サブアドレスとは別の機能であり、メール運用を正確に理解するためには両者を区別する必要がある

**Rubyistへの影響:** メール機能を使用する際にサブアドレスとエイリアスを混同しないことで、より正確で安全なメール運用が可能になる

### ★★★☆☆ [Claude Code と Codex のレビュー機能は脆弱性をどれだけ見つけられるかをOWASP Benchmark で検証](https://zenn.dev/yukkie1114/articles/3d927e8c28e085) `セキュリティ` `コードレビュー` `owasp` `claudecode` `codex`

- Claude CodeとCodexのレビュー機能4種類をOWASP Benchmarkで脆弱性検知力を測定。Codex /reviewが0.964で最高スコア、Claude Code /security-reviewが0.945で次点。
- 検出率ではClaude Code /security-reviewが98.2%で最高だが、誤検知率3.6%を考慮するとCodex /reviewの検出率96.4%・誤検知率0%が最適バランス。
- 測定対象は正解ラベル付きの110件データで、サブスク内で使えるスラッシュコマンドのみ。GitHub連携の自動レビュー機能は対象外。

**Rubyistへの影響:** Ruby開発者には直接影響なし。ただしAIレビューの精度比較知見は他言語開発にも参考になる可能性あり。

### ★★☆☆☆ [デュアルブート備忘録](https://zenn.dev/qnighy/articles/09136afbf27bf3) `linux` `windows` `uefi` `lvm`

- 著者が4年ぶりにWindows/Linuxのデュアルブート構成を復活させた。WSLのDocker連携不安定性とsystemd未対応が理由で、Linuxドライバサポート改善に期待。

- 開発環境遍歴：Cygwin→デュアルブート→VM(Ubuntu)→WSL移行。WSLの限界を感じ、セキュアブートと暗号化を維持したままのデュアルブート再構築を記録。
