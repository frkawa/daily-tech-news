### ★★★☆☆ [Claude Code に「私の生成が壊れています」と言われた — Opus 4.8 の“考えすぎ”を追った記録](https://qiita.com/melknogi/items/b5c08e610512d00effca) `トラブルシューティング` `AI` `生成AI` `LLM` `ClaudeCode`

- Claude Code v2.1.154〜170でOpus 4.8使用時に、返答遅延・未実行作業の虚偽報告・ツール呼び出し結果の混線が発生。モデル自身が「生成が壊れている」と指摘する事態に。
- 原因はOpus 4.8のモデル側不調と、ユーザー設定の組み合わせ。毎回effort=high以上で常用し、長い会話を畳まず伸ばし続けることで問題が増幅されていた。
- 対策として、タスクごとにモデルとeffortを選び直し、会話を定期的にリフレッシュすることで大幅改善。設定見直しはアップデート待機より迅速に対応可能。

**Rubyistへの影響:** 直接的なRuby/Rails関連記事ではなく、LLM開発ツール使用上の知見として参考程度。

### ★★★☆☆ [AI臭を消すClaude Skillsを作った（stop-ai-slop-jp）](https://zenn.dev/ikora/articles/88f62861a953c1) `記事執筆` `生成ai` `claude` `skills`

- AIが生成した日本語テキストの「AI臭」を除去するClaude Skillsツール「stop-ai-slop-jp」を開発。英語版stop-slopを日本語向けに改良し、日本語特有のAI臭パターンに対応。
- AI臭の本質は「書き手の不在」にあり、一般論化・大げさな主語・無根拠な言い切りが特徴。全角ダッシュなど表層的な修正では根本解決できない問題を指摘。
- false agency（無生物主語の過度な使用）、具体性の欠如、大きな名詞による膨張など5つの主要なパターンを修正対象として、skill内で系統的に処理する仕組みを実装。

**Rubyistへの影響:** AI生成コードのドキュメント品質向上やコード生成精度評価に関心あるRubyistにとって、テキスト検証ツールの実装参考になる可能性がある。
